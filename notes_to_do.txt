Security Issues(after we've sorted out the above)(Don't work on these yet, just brainstorm)
1. How do we make sure we're not snapshot and storing sensitive data?
   → Filter at event collection: Don't capture URLs/page content from sensitive domains (banking, healthcare, etc.). Hash/redact PII before snapshot. Detect keywords like "password", "credit card", SSN patterns.

2. If we do track sensitive data, how do we protect it?
   → Encrypt at-rest (SQLite encryption). Encrypt in-transit (HTTPS for API). Access control (only user owns their data). Consider zero-knowledge architecture where snapshots are generated client-side, not sent to backend.

3. What if the sensitive data is needed for the specific session?
   → This is the core tension. Solutions: (a) User opt-in for sensitive sites, (b) Local-only processing (client snapshots), (c) Separate encrypted vault for sensitive sessions, (d) Retention limits (auto-delete after X days).
